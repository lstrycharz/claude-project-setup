# Recurring Report — Validation Checklist

Claude runs this checklist against the report before declaring it done. Each item is a yes/no question — if any item fails, that section gets reworked.

This checklist is what turns the rules file from passive reference material into active enforcement. Aim for 15–25 items.

## Structure and required sections

- [ ] Executive summary is at the top, before any detail
- [ ] All required sections from `report-rules.md` are present, in the prescribed order
- [ ] No section is missing or out of order

## Content quality

- [ ] Every metric reported includes appropriate context (MoM, YoY, vs. target — per team rules)
- [ ] Anomalies are surfaced before raw numbers
- [ ] Recommendations include direction, magnitude, and owner
- [ ] Tone matches the audience defined in the rules file

## Conventions

- [ ] Visualization rules followed (chart vs. table choices, formatting)
- [ ] Voice and tone match `references/global/general-rules.md`
- [ ] No forbidden words present
- [ ] All factual claims supported by data referenced in the report

## Polish

- [ ] No placeholders or `[TBD]` markers remain
- [ ] All numbers cross-checked against source data
- [ ] Length within team-standard range
- [ ] Headings and formatting consistent with team conventions

---

*Add or remove items as your team's standards evolve. Each item should be a clear yes/no question — anything that requires interpretation isn't a checklist item, it's a rule for the rules file.*
